# ShowRunner — Monmouthshire Show 2026

Mobile-first coordinator dashboard for the **Band Stand** at the Monmouthshire Show, 16 August 2026.

**Stack:** Hono.js · Drizzle ORM · Cloudflare Pages + Pages Functions · D1 (SQLite) · esbuild · TypeScript strict

---

## Quick start (local dev)

```bash
npm install

# Copy secrets template
cp .dev.vars.example .dev.vars
# Edit .dev.vars with your Gmail OAuth credentials (Phase 5)

# Run DB migrations locally
npm run db:migrate:local

# Seed with show data
npm run db:seed:local

# Start dev server (esbuild watch + wrangler pages dev)
npm run dev
# → http://localhost:8788
```

## Cloudflare setup

### 1. Create D1 database

```bash
wrangler d1 create monmouth-show-db
```

Copy the `database_id` from the output into:
- `wrangler.toml` (Pages project)
- `workers/gmail-sync/wrangler.toml` (cron Worker)

### 2. Apply migrations

```bash
# Local (for dev)
npm run db:migrate:local

# Remote (production)
npm run db:migrate:remote
```

### 3. Seed data

```bash
npm run db:seed:local   # local dev
npm run db:seed:remote  # production
```

### 4. Deploy Pages app

```bash
npm run deploy
```

Configure the Pages project in the Cloudflare dashboard:
- **Build command:** `npm run build`
- **Build output dir:** `public`
- **Root:** `/` (repo root)
- **D1 binding:** `DB` → `monmouth-show-db`

### 5. Deploy Gmail sync Worker

```bash
wrangler deploy --config workers/gmail-sync/wrangler.toml
```

Set secrets on the Worker:

```bash
wrangler secret put GMAIL_CLIENT_ID     --config workers/gmail-sync/wrangler.toml
wrangler secret put GMAIL_CLIENT_SECRET --config workers/gmail-sync/wrangler.toml
wrangler secret put GMAIL_REFRESH_TOKEN --config workers/gmail-sync/wrangler.toml
```

## Gmail OAuth — one-time setup (Phase 5 detail)

> **This must be done by Dan on his own machine** so the consent screen uses his Google account.

1. In [Google Cloud Console](https://console.cloud.google.com/):
   - Create a project (or reuse one)
   - Enable the **Gmail API**
   - Create OAuth 2.0 credentials → *Desktop app*
   - Download the client secret JSON; note `client_id` and `client_secret`

2. Set env vars and run the consent script (**Phase 5 — not yet implemented**):

   ```bash
   GMAIL_CLIENT_ID=xxx GMAIL_CLIENT_SECRET=yyy npm run oauth:gmail
   ```

   When implemented in Phase 5, the script will open a local server, print a consent URL, and on success output the `wrangler secret put` commands to store the refresh token. Until then, follow the manual OAuth steps in Google's documentation and store the refresh token via `wrangler secret put GMAIL_REFRESH_TOKEN`.

3. The refresh token is stored as a Cloudflare secret — never committed to git.

4. The sync Worker refreshes the access token automatically on each cron run.

**Scope used:** `https://www.googleapis.com/auth/gmail.readonly` (read-only).

## Gmail filter settings

Stored in D1 `settings` table, editable in the in-app Settings page:

| Key | Default | Description |
|-----|---------|-------------|
| `gmail_label` | `MonShow` | Gmail label Dan applies to relevant threads |
| `gmail_contacts` | *(see seed)* | Comma-separated email addresses to also include |

The cron Worker builds a Gmail query like:
```text
label:MonShow OR from:atcstephharris@gmail.com OR from:...
```
